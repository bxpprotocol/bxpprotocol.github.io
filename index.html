What BXP defines

A format layer,
not a platform.

BXP is a pure software protocol standard. Like HTTP, PDF, and MP3 — it requires no proprietary hardware, no centralised infrastructure, and no licensing fees.

It runs on any device, any platform, any geography, at any scale. A sensor company in Japan can write their own C++ implementation. A Nigerian startup builds their own node. Nobody needs permission.

The spec lives on GitHub permanently. The standard is the law. Everything else is an implementation of it.

  • Universal .bxp File Format
    Any device or software reads and writes it. Covers 31 atmospheric agents including PM2.5, NO2, ozone, VOCs, heavy metals, and biological agents.
  • BXP_HRI Health Risk Index
    Composite health risk score 0–100. WHO-derived weighting across all available agents. Single actionable number for any exposure record.
  • REST API Specification
    Complete federated node architecture. Any institution runs a node. Nodes interoperate via standard endpoints. Nobody owns the network.
  • Privacy Framework
    Individual records private by default. AES-256-GCM encryption. SHA-256 hashed identifiers. GDPR compliant by design.
